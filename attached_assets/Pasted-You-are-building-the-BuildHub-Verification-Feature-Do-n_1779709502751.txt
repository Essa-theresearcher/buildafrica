=======================================================

Create these tables in Supabase with exact column names:

TABLE: verification_applications
- id: uuid, primary key, default gen_random_uuid()
- builder_id: uuid, foreign key → profiles.id
- status: text, default 'pending'
  allowed values: 'pending', 'challenge_assigned', 
  'submitted', 'call_scheduled', 'passed', 
  'failed', 'conditional_pass'
- primary_skill: text
  allowed values: 'frontend', 'backend', 
  'fullstack', 'mobile', 'data', 'devops'
- challenge_id: uuid, nullable, 
  foreign key → challenges.id
- challenge_assigned_at: timestamptz, nullable
- challenge_deadline: timestamptz, nullable
- submission_github_url: text, nullable
- submission_demo_url: text, nullable
- submission_explanation: text, nullable
- submission_screenshot_url: text, nullable
- submitted_at: timestamptz, nullable
- call_scheduled_at: timestamptz, nullable
- call_completed_at: timestamptz, nullable
- admin_notes: text, nullable
- admin_checklist: jsonb, nullable
- decision_made_at: timestamptz, nullable
- created_at: timestamptz, default now()

TABLE: challenges
- id: uuid, primary key, default gen_random_uuid()
- title: text, not null
- skill_category: text, not null
  allowed values: 'frontend', 'backend', 
  'fullstack', 'mobile', 'data', 'devops'
- description: text, not null
- requirements: jsonb, not null
  (array of strings, each is a specific requirement)
- local_context: text, not null
  (explains the East African real-world scenario)
- time_limit_hours: integer, default 72
- is_active: boolean, default true
- created_at: timestamptz, default now()

TABLE: builder_badges
- id: uuid, primary key, default gen_random_uuid()
- builder_id: uuid, foreign key → profiles.id
- badge_type: text, not null
  allowed values: 'verified_builder', 
  'shipped_project', 'strong_explainer',
  'reliable_collaborator', 'coffee_code_member'
- badge_scope: text, nullable
  (example: 'frontend only' or 'backend only')
- granted_by: uuid, foreign key → profiles.id
- granted_at: timestamptz, default now()
- is_active: boolean, default true

TABLE: admin_call_checklist
- id: uuid, primary key, default gen_random_uuid()
- application_id: uuid, 
  foreign key → verification_applications.id
- can_demo_without_hesitation: boolean, default false
- can_explain_approach: boolean, default false
- answered_unexpected_questions: boolean, default false
- made_live_change: boolean, default false
- commit_history_real: boolean, default false
- has_other_projects: boolean, default false
- knows_community: boolean, default false
- gut_feeling_pass: boolean, default false
- total_checked: integer, 
  generated always as (
    (can_demo_without_hesitation::int +
     can_explain_approach::int +
     answered_unexpected_questions::int +
     made_live_change::int +
     commit_history_real::int +
     has_other_projects::int +
     knows_community::int +
     gut_feeling_pass::int)
  ) stored
- admin_id: uuid, foreign key → profiles.id
- completed_at: timestamptz, default now()

Write the SQL for all 4 tables.
Include Row Level Security policies:
- Builders can only read their own application
- Builders can insert one application per profile
- Builders can update their own application 
  only when status is 'challenge_assigned'
  (this is when they submit their work)
- Admins can read and update all applications
- Challenges table is readable by all authenticated users
- builder_badges table is readable by everyone
- Only admins can insert into builder_badges
