You are adding a Startup Showcase and Paid Marketing 
feature to BuildHub.

BuildHub is a builder discovery platform.
Startups are one type of builder on the platform.
They are not a separate product.
They live inside BuildHub under /startups

There are two layers being built here:

LAYER 1 — Free Showcase (every startup gets this)
LAYER 2 — Paid Marketing (startups pay to get amplified)

Do not build investor profiles yet.
Do not build payment processing yet.
Build the UI and logic for both layers.
Mark paid features clearly in the UI.
When a startup tries to activate a paid feature:
Show a "Coming Soon — Join Waitlist" modal
that captures their email into a waitlist table.
This lets you validate demand before building payments.

=======================================================
UNDERSTAND THE BUSINESS MODEL FIRST
=======================================================

FREE TIER — What every startup gets at no cost:
- A startup profile page
- Listed in the startups directory
- Ability to post building updates
- Team member showcase
- Basic contact button
- Upvote button for community support

PAID TIER — What startups pay for:
- Homepage spotlight placement (7 days)
- "Featured" badge and priority in directory
- Startup of the Week (newsletter + social push)
- Boosted visibility in search results
- "Investor Ready" badge (future — V2)

The paid tier is your first revenue stream.
Build the UI for it completely.
Replace the payment action with a waitlist capture.
This proves demand before you build Stripe.

=======================================================
STEP 1 — DATABASE TABLES
=======================================================

TABLE: startups
- id: uuid, primary key, default gen_random_uuid()
- founder_id: uuid, foreign key → profiles.id
- name: text, not null
- slug: text, not null, unique
- tagline: text, not null, max 100 characters
- description: text, not null
- problem_solved: text, not null
- category: text, not null
  allowed values: 'fintech', 'edtech', 
  'healthtech', 'logistics', 'saas', 
  'agritech', 'ecommerce', 
  'developer_tools', 'other'
- stage: text, not null, default 'building'
  allowed values: 'building', 'launched', 'growing'
- logo_url: text, nullable
- cover_image_url: text, nullable
- demo_url: text, nullable
- product_url: text, nullable
- github_url: text, nullable
- looking_for: jsonb, nullable
- traction_users: text, nullable
  allowed values: 'pre-launch', '1-100',
  '100-500', '500-1000', '1000-5000', '5000+'
- traction_revenue: text, nullable
  allowed values: 'pre-revenue', 'early-revenue',
  'growing-revenue', 'prefer-not-to-say'
- markets_served: text[], nullable
- is_hiring: boolean, default false
- is_verified: boolean, default false
- is_active: boolean, default true

-- FREE TIER FLAGS
- is_listed: boolean, default true

-- PAID TIER FLAGS  
- is_featured: boolean, default false
- featured_until: timestamptz, nullable
- is_spotlight: boolean, default false
- spotlight_until: timestamptz, nullable
- is_startup_of_week: boolean, default false
- startup_of_week_date: date, nullable
- is_boosted: boolean, default false
- boosted_until: timestamptz, nullable

-- MARKETING METADATA
- marketing_tier: text, default 'free'
  allowed values: 'free', 'featured', 
  'spotlight', 'premium'
- total_upvotes: integer, default 0
- total_views: integer, default 0
- created_at: timestamptz, default now()
- updated_at: timestamptz, default now()

TABLE: startup_team_members
- id: uuid, primary key, default gen_random_uuid()
- startup_id: uuid, foreign key → startups.id
- builder_id: uuid, foreign key → profiles.id
- role: text, not null
- is_founder: boolean, default false
- joined_at: timestamptz, default now()

TABLE: startup_updates
- id: uuid, primary key, default gen_random_uuid()
- startup_id: uuid, foreign key → startups.id
- posted_by: uuid, foreign key → profiles.id
- title: text, not null
- body: text, not null
- update_type: text, not null
  allowed values: 'shipped', 'milestone',
  'setback', 'looking_for', 'general'
- created_at: timestamptz, default now()

TABLE: startup_upvotes
- id: uuid, primary key, default gen_random_uuid()
- startup_id: uuid, foreign key → startups.id
- user_id: uuid, foreign key → profiles.id
- created_at: timestamptz, default now()
- UNIQUE constraint on (startup_id, user_id)

TABLE: startup_views
- id: uuid, primary key, default gen_random_uuid()
- startup_id: uuid, foreign key → startups.id
- viewer_id: uuid, nullable (null if not logged in)
- viewed_at: timestamptz, default now()

TABLE: marketing_waitlist
- id: uuid, primary key, default gen_random_uuid()
- email: text, not null
- startup_id: uuid, nullable, 
  foreign key → startups.id
- startup_name: text, nullable
- package_interest: text, not null
  allowed values: 'featured', 'spotlight',
  'startup_of_week', 'premium'
- message: text, nullable
- created_at: timestamptz, default now()

TABLE: marketing_packages
- id: uuid, primary key, default gen_random_uuid()
- name: text, not null
- slug: text, not null, unique
- tagline: text, not null
- description: text, not null
- features: jsonb, not null
- price_usd: integer, not null
- duration_days: integer, not null
- is_active: boolean, default true
- sort_order: integer, default 0

RLS POLICIES:
- startups readable by everyone
- founders can insert and update own startup
- startup_updates readable by everyone
- only founder can insert updates for own startup
- startup_upvotes: one per user per startup
- startup_views: anyone can insert
- marketing_waitlist: anyone can insert
- marketing_packages: readable by everyone
- admin can update all tables
